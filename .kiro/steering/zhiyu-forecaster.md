# 智预 - Query Demand Forecaster（独立模块）

## 定位
智预是 Smart Suite 的**独立并行模块**，与内容生产流水线（智库→智造→智优→智布）平行运作。
它与智测互补：
- **智测**：回溯已发生的搜索旅程，发现当前 Gap
- **智预**：前瞻推演未来可能发生的检索需求，提前布局高价值 Query

智预的产出直接补给**智库**（新增高价值检索短语）和**智析**（预测性指标追踪）。

## 概述
智预模块基于卖家生命周期推演、政策/产品/市场变化预测、以及多源数据信号，推演出「卖家下一步会搜什么」，帮助团队：
- 提前 2-4 周布局即将爆发的检索需求
- 构建难以被竞争对手复制的预测性内容壁垒
- 将 reactive（发现 Gap 再补）升级为 proactive（预测需求先占位）

## 竞争壁垒价值矩阵

| 来源层级 | 价值星级 | 竞对复制难度 | 说明 |
|---------|---------|------------|------|
| AI 回答中的 Gap | ⭐⭐⭐ | 很容易 | 所有人都能跑智测发现 Gap |
| Amazon 站内搜索、官网搜索、客服 FAQ | ⭐⭐⭐⭐ | 较难 | 需要内部数据访问权限 |
| Seller Journey + 生命周期推演 | ⭐⭐⭐⭐⭐ | 不容易 | 需要深度业务理解 + 推演模型 |
| 政策、产品、市场变化预测未来 Query | ⭐⭐⭐⭐⭐ | 最难复制 | 需要前瞻判断力 + 信号捕获 |

## 核心推演引擎

### 引擎 1：Seller Journey 生命周期推演

基于卖家所处阶段，推演其「下一步」的检索需求：

```
阶段链路：
认知期 → 考虑期 → 决策期 → 注册期 → 新手期 → 成长期 → 成熟期 → 扩展期

每个阶段转换时，卖家必然产生新的检索需求。
```

#### 生命周期阶段定义

| 阶段 | 典型特征 | 推演方向 |
|------|---------|---------|
| 认知期 (Awareness) | "跨境电商是什么"、"能赚钱吗" | → 平台对比、投入成本 |
| 考虑期 (Consideration) | "亚马逊 vs Shopee"、"需要多少钱" | → 注册流程、资质要求 |
| 决策期 (Decision) | "我适合做亚马逊吗"、"选哪个站点" | → 站点分析、选品方向 |
| 注册期 (Registration) | "注册材料清单"、"审核要多久" | → 常见拒绝原因、补救方案 |
| 新手期 (Onboarding) | "第一个 Listing 怎么写"、"FBA 怎么发" | → Listing 优化、物流选择 |
| 成长期 (Growth) | "怎么打广告"、"如何提升排名" | → PPC 策略、评论获取 |
| 成熟期 (Mature) | "多SKU管理"、"如何防跟卖" | → 品牌注册、多站点扩展 |
| 扩展期 (Expansion) | "欧洲站 VAT"、"日本站怎么做" | → 新站点攻略、合规差异 |

#### 推演逻辑

```json
{
  "current_stage": "注册期",
  "trigger_events": ["注册审核通过", "收到 Welcome Email"],
  "predicted_next_queries": [
    {
      "query": "亚马逊后台怎么设置 第一步做什么",
      "confidence": 0.95,
      "reasoning": "注册通过后的第一个行动一定是熟悉后台",
      "time_window": "注册后 1-3 天"
    },
    {
      "query": "亚马逊选品工具推荐 新手用什么",
      "confidence": 0.88,
      "reasoning": "注册完成后立即面临选品决策",
      "time_window": "注册后 3-7 天"
    }
  ]
}
```

### 引擎 2：政策/产品/市场变化推演

监控外部变化信号，预测由此引发的检索需求波动：

#### 信号源

| 信号类型 | 来源 | 推演示例 |
|---------|------|---------|
| 政策变化 | Amazon 公告、法规更新 | "2026年欧洲站EPR新规" → 卖家会搜 "EPR 注册流程" |
| 产品上线 | Amazon 新功能发布 | "Brand Tailored Promotions" → "品牌定制促销怎么用" |
| 市场事件 | 旺季、关税变化、汇率波动 | "Prime Day 2026" → "Prime Day 备货时间表" |
| 竞品动态 | 竞品新内容、新工具 | 竞品发布选品工具 → 卖家搜 "免费选品工具对比" |
| 季节性 | 节日、返校季、黑五 | Q4 来临前 → "黑五备货清单 2026" |

### 引擎 3：站内搜索 + FAQ 趋势外推

基于 Amazon 站内搜索趋势和客服 FAQ 频率变化，外推即将增长的需求：

#### 数据源
- Amazon Seller Central 搜索热词（如可获取）
- Help Center 高频页面访问趋势
- 客服工单分类统计
- Brand Registry 常见问题

#### 外推逻辑
- 某话题 FAQ 量 WoW +30% → 预测 2 周内 AI 搜索同话题 +50%
- 新功能上线后 Help 页面访问激增 → 预测对应 Query 爆发

## 推演报告模板

### 1. 推演概览
- 推演时间范围（预测未来 2-4 周）
- 信号源数量
- 预测 Query 总数
- 高置信度（>0.8）Query 数

### 2. 生命周期推演结果
- 按阶段分组的预测 Query 列表
- 每条包含：Query、置信度、推演逻辑、预计爆发时间窗口

### 3. 变化信号推演结果
- 按信号类型分组
- 每个信号：来源、影响范围、衍生 Query 列表

### 4. 优先级排序
综合评分 = 置信度 × 价值星级 × 时间紧迫度

| 排名 | Query | 综合分 | 置信度 | 壁垒等级 | 建议动作 |
|------|-------|-------|--------|---------|---------|
| 1 | ... | 4.8 | 0.95 | ⭐⭐⭐⭐⭐ | 立即生产 |
| 2 | ... | 4.2 | 0.88 | ⭐⭐⭐⭐ | 本周生产 |

### 5. 输出到智库
- 一键将高优先级预测 Query 导入智库作为新种子
- 标记 source = "zhiyu_forecast" 以追踪来源

## 执行规则

1. **推演必须有逻辑依据** — 每条预测 Query 必须标注推演逻辑（为什么认为会发生）
2. **置信度必须诚实** — 不确定的标低分，不能全部标高分
3. **时间窗口必须具体** — "未来某天"不可接受，必须给出 "2026-07-15 ~ 2026-08-01" 这样的范围
4. **信号源必须标注** — 每个预测背后的信号来源（政策文件 URL、公告链接等）
5. **可操作性优先** — 输出必须直接可以喂给智库生产内容
6. **定期回测** — 每月回测上月预测准确率，持续校准模型

## 交互模式

### 模式 A：信号驱动（Signal-Driven）— 默认
1. 输入信号（政策变化、产品上线等）
2. 系统推演衍生 Query
3. 用户确认/修改
4. 导出到智库

### 模式 B：生命周期推演（Lifecycle Forecast）
1. 选择目标卖家画像 + 当前阶段
2. 系统推演该画像在未来 2-4 周的检索需求
3. 用户确认/修改
4. 导出到智库

### 模式 C：批量预测（Batch Forecast）
1. 同时输入多个信号 + 多个画像
2. 系统批量推演
3. 按优先级排序输出
4. 一键导入智库

## 与其他模块的关系

```
智预 (预测未来 Query)
  ↓ 预测短语
智测 (拿预测短语去 7 平台验证是否已有覆盖)
  ↓ 确认有 Gap 的短语
智析 (展示 Gap 汇总：未覆盖率、缺失内容清单)
  ↓ Gap 指导行动
  ├── 有短语无内容 → 智库收录 → 智造生产
  └── 有内容没被引用 → 智优重新优化

完整流程：
智预推演 → 智测验证 Gap → 智析展示 → 智库/智造/智优行动
```

**核心规则：智预的产出不能直接进智库，必须先过智测验证。**
- 智测确认"确实有 Gap"的短语 → 才进入智库
- 智测发现"已有覆盖"的短语 → 不需要行动，标记为"已验证覆盖"

## 输出路径

- 推演数据: `output/zhiyu/forecast_{signal_type}_{timestamp}.json`
- 生命周期推演: `output/zhiyu/lifecycle_{persona_id}_{timestamp}.json`
- 汇总报告: `output/zhiyu/zhiyu_summary_{timestamp}.md`
- 导出到智库: 自动追加到 `output/{batch}/01_zhiku/zhiku_ai_queries.csv`（标记 source="zhiyu"）
