### 页面结构（单页线性）
```
① 生成检索短语
   ├ 源 A：SEO/SEM 关键词裂变（上传 CSV）

   └ 源 B：核心词根裂变（text_area 多个词根）
       ├ 预估数量（词根数×15）
       ├ 进度条
       └ [🚀 裂变源 B] / [✅ 源 B 已完成]
   说明：可选 A、B 单独或同时使用，结果自动合并去重

② 短语库 — 分类 & 校对

📊 类别覆盖看板（第②步之后展示）
   ├ KPI：总短语 | 已覆盖类别/35 | 空类别 | 已选中
```

### 关键逻辑
- **两源独立执行** — 各有自己的按钮和进度，互不干扰
- **自动去重** — 系统在 append 时自动基于 ai_query 去重
- **自动分类** — AI 裂变时自动分配 35 个类别
- **累计追加** — 每次裂变追加到已有数据，不覆盖
- **Offset 模式** — 源 A 跳过已处理关键词，处理新的
- **源 B 完成后隐藏按钮** — 显示 ✅ 状态
- **CSV 引号规则** — 含逗号字段必须双引号包裹
- **类别名无逗号** — "怎么做跨境电商及流程费用了解"（去掉中文逗号）

### 评分维度
- **综合分** (priority_score): 1-5，与业务转化相关性
- **AI 适配分** (ai_fit_score): 1-5，是否符合 AI 搜索输入习惯
- **适用平台** (target_market): CN / WW / Both
  - CN = 国内 AI 平台极简口语（DeepSeek/豆包/Kimi）
  - WW = ChatGPT/Perplexity 稍完整中文提问
  - Both = 通用
- 所有短语统一中文（服务 CN base seller）

- 源 B：每个词根生成 15 条
- 每批最多 50 个关键词（可选 10/20/30/50）

## 已知 Bug 需修复
1. engine.py `run_zhiku` 写文件必须用 append 模式不能覆盖
2. CSV 含逗号字段需双引号（prompt 里已加规则）
3. `load_csv_safe` 需要 `on_bad_lines='skip'`
4. 进度条需实时从文件读取（不依赖页面初始化时的 df_q）
5. `st.rerun()` 在写文件后不要立即调用（会导致数据丢失）
6. 不要在 app.py 里做二次去重（让 engine 内部处理）
7. 智库页面每次打开时操作面板清零（历史数据保留在底部历史记录中）
   - 第①步：不自动读取旧文件显示进度，每次都是全新状态
   - 第②步：只显示本次 session 生成的数据
   - 历史数据（之前的 CSV 文件）在底部"历史记录" expander 中可查看下载

## 技术注意事项
- 使用 `st.session_state["jump_to_page"]` + 设置 radio key 实现页面跳转
- 不要用 `st.stop()` 在 tab 外面（会阻止其他 tab 渲染）
- 批量替换文件内容时不要用 Python 单行 replace（容易破坏文件编码）
