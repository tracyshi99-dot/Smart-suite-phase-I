# 检索短语准确度交叉验证规则

## 概述
本文档定义所有用于验证检索短语准确度的方法。每条短语可以通过多种方法交叉验证，最终得到综合准确度评分。

**核心原则：单一验证方法不可靠，必须多维度交叉验证。**

---

## 验证维度总览

短语准确度从以下 **3 个维度** 交叉验证 + 1 个发现动作：

```
┌─────────────────────────────────────────────────────┐
│  第一部分：短语精准度（产出 + 校准 + 去重）          │
│                                                     │
│  A 来源真实性 → B 校准 → 去重聚类                    │
│  目标：得到一批精准、不重复、业务相关的检索短语       │
└─────────────────────────────────────────────────────┘
                        ↓
┌─────────────────────────────────────────────────────┐
│  第二部分：Gap 发现（覆盖验证）                      │
│                                                     │
│  C 把精准短语全量送智测 → 发现内容/链接缺口          │
│  目标：找到需要行动的 Gap → 指导智造/智优            │
└─────────────────────────────────────────────────────┘
```

---

# 第一部分：短语精准度

**目标：从各种来源产出短语 → 校准精度 → 去重整理 → 得到精准短语库**

---

### 维度 A：来源真实性（产出短语 — 这个问句是不是真实用户在问的）

| # | 方法 | 可靠度 | 来源 |
|---|------|--------|------|
| A1 | AI 下拉联想 + "大家还在问" | 95% | 7 款 AI 平台搜索建议 |
| A2 | 逆向召回（问 AI：什么问句能找到这个内容） | 92% | 7 款 AI 平台反向询问 |
| A3 | AI 问答社区原生提问 | 90% | 知乎直答、Perplexity 历史提问 |
| A4 | 官方渠道搜索词（站内搜索/官网/客服FAQ/微信公众号） | 85% | Amazon 站内、Seller Central、Help Center、公众号后台 |
| A5 | SEO/SEM 核心关键词扩展 | 82% | Google Ads、百度竞价词、SEO 关键词工具裂变 |
| A6 | 跨境社群/客服/直播真实问句 | 80% | 卖家社群、客服工单、直播评论 |
| A7 | 谷歌 AlsoAsked / AnswerThePublic | 75% | Google PAA、海外问句工具 |
| A8 | AI 批量扩写（句式改写 + 词根扩词） | 60% | 大模型生成（仅做兜底补充） |

**逻辑：** 来源越接近真实用户的原始输入 → 准确度越高

---

### 维度 B：校准（精炼短语 — 多角度确认短语是否精准可用）

对维度 A 产出的短语，从 3 个子维度交叉校准：

#### B1：跨平台共识校准

| # | 方法 | 说明 |
|---|------|------|
| B1a | 跨平台回答一致性 | 同一问句在 7 平台的回答核心要点是否一致（一致=精准，分歧大=模糊） |
| B1b | 来源交叉计数 | 该问句被几种不同来源（A1-A7）独立产出过（多来源=高可信） |

#### B2：35 类别匹配校准

| # | 方法 | 说明 |
|---|------|------|
| B2a | 类别归属 | 每条短语必须能归入 35 个内容类别中的至少 1 个 |
| B2b | 意图匹配 | 短语意图是否与该类别的内容方向一致 |
| B2c | 无法归类 → 丢弃或新建类别 | 无法匹配任何现有类别的短语 = 不在业务范围 |

**35 类别来自 `content-rules.md`：** 跨境电商知识、注册、选品、物流、广告、合规等

#### B3：时效性校准

| # | 方法 | 说明 |
|---|------|------|
| B3a | 年份检查 | 含过期年份（如2024/2025）→ 更新年份或丢弃 |
| B3b | 政策/功能存续 | 涉及已下线功能/已取消政策 → 丢弃 |
| B3c | 季节性窗口 | 季节性话题当前不在有效期 → 标记"待激活"暂不使用 |

**校准结果：**
- 全部通过（B1+B2+B3）→ 精准短语，进入去重
- B1 不通过（平台共识低）→ 降权，标记为"平台特异性短语"
- B2 不通过（无法归类）→ 丢弃
- B3 不通过（过期）→ 丢弃或更新后重新校准

---

### 去重聚类（第一部分最后一步）

对校准通过的短语做整理，**避免重复短语浪费 7 平台验证资源**：

1. **完全重复** → 删除，保留最早入库的
2. **近义重复**（语义相似 > 85%）→ 合并为一条，取更口语/更完整的版本
3. **同主题不同角度** → 两条都保留
4. **按频次排序** — source_count 高的排前面
5. **按 35 类别分组** — 对应到具体类别，方便后续按类别批量验证和生产

**去重完成 = 精准短语库就绪 → 进入第二部分**

---

# 第二部分：Gap 发现

**目标：把精准短语全量送智测验证，发现内容/链接缺口，指导智造/智优行动**

### 维度 C：Gap 发现（基于精准短语，验证我方内容覆盖状态）

**前提：** 只有通过第一部分（来源真实 + 校准通过 + 去重后）的精准短语才进入维度 C

**操作：** 把精准短语全量输入 7 个 AI 平台搜索，检查回答中我方内容的覆盖情况

| 短语类型 | 检查内容 | Gap 判定 |
|---------|---------|---------|
| 品牌类短语 | ① 有没有官方品牌提及 ② 有没有官网链接 | 缺任一 = Gap |
| 行业类短语 | ① 有没有官方提及 | 无提及 = Gap |

**Gap 发现后的行动路由：**
| 状态 | 行动 |
|------|------|
| 完全覆盖（品牌+链接都有） | 不需行动 → 智析记录 |
| 部分 Gap（有提及没链接 / 有链接没提及） | → 智优优化现有内容 |
| 完全 Gap（没提及没链接） | → 智造生产新内容 |

**规则：** 全量执行，不允许抽样。所有精准短语必须 100% 验证后才能决定下一步。

## 方法 1：逆向召回验证

**详见 `reverse-recall-rules.md`**

**操作：** 把被 AI 抓取的内容放到 7 平台，问"什么问句能找到这个内容"
**输出：** 每条问句的跨平台共识度
**判定标准：**
- 5-7 平台共识 → 准确度 95%+
- 3-4 平台共识 → 准确度 85%
- 1-2 平台共识 → 准确度 70%

---

## 方法 2：AI 下拉联想匹配验证

**操作：** 在每个 AI 平台的搜索/对话框里输入词根，收集下拉建议和"相关问题"

**执行步骤：**
1. 打开 AI 平台（无痕模式）
2. 输入词根（如"亚马逊FBA"）
3. 记录下拉联想词（在输入过程中平台自动建议的问句）
4. 提交后记录回答底部的"大家还在问"/"相关问题"
5. 在 7 平台重复

**验证规则：**
- 如果一条短语出现在平台的下拉联想中 → 该平台标记为 `dropdown_hit = TRUE`
- 如果一条短语出现在"大家还在问"中 → 该平台标记为 `related_hit = TRUE`

**准确度判定：**
- 在 5+ 平台的下拉中出现 → 准确度 95%（真实高频问句）
- 在 3-4 平台下拉中出现 → 准确度 88%
- 仅在 1-2 平台下拉中出现 → 准确度 75%
- 不在任何平台下拉中 → 需要其他方法交叉验证

---

## 方法 3：AI 问答社区原生提问校准

**来源：** 知乎直答、悟空问答、Perplexity 历史提问、AI 助手公开对话帖

**操作步骤：**
1. 在知乎搜索"亚马逊开店""FBA""跨境电商"等词根
2. 收集"知乎直答"中的用户原始提问（不是答案）
3. 在 Perplexity 的 Discover/Trending 中找跨境电商相关提问
4. 在 AI 助手社区（如 ChatGPT 公开分享帖）找真实对话中的用户提问

**筛选规则：**
- 只保留跨境电商/亚马逊相关问句
- 排除纯讨论帖（非提问形式的不要）
- 排除已有明确答案且不需要更新的老问题

**准确度判定：**
- 问句直接来自用户原始输入 → 90%
- 问句经过编辑/整理 → 降至 80%

**去重规则：** 如果与已有短语语义重复率 > 80%，合并到已有短语并增加频次计数

---

## 方法 4：官方渠道搜索词校准

**来源：** Amazon 站内搜索、官网搜索、Seller Central 搜索、客服 FAQ 高频词、微信公众号后台搜索词

**操作步骤：**
1. 从 Amazon 官网/Seller Central 站内搜索框收集下拉建议词
2. 从官网 Help Center / FAQ 提取高频被搜索的问题
3. 从客服工单系统提取卖家高频咨询问题
4. 从微信公众号后台"搜一搜"关键词数据、文章搜索来源词提取
5. **关键步骤：** 将官方渠道搜索词改写为 AI 口语疑问句格式

**改写规则：**
| 原始广告搜索词 | 改写为 AI 问句 |
|---------------|---------------|
| FBA费用 | "亚马逊FBA费用怎么算 新手第一批货要花多少钱" |
| 欧洲站VAT | "欧洲站VAT税怎么申报 中国卖家需要注意什么" |
| Listing优化 | "亚马逊Listing怎么写才能排名靠前" |

**改写原则：**
- 加上疑问词（怎么/如何/多少/哪些/为什么）
- 加上场景限定（新手/中国卖家/2026年）
- 保持口语化（像用户在对话框里打字的样子）

**准确度判定：**
- 原始搜索词 5+ 词且高搜索量 → 改写后准确度 85%
- 原始搜索词 3-4 词中等搜索量 → 改写后准确度 75%
- 纯短词（1-2词）改写 → 准确度 65%（降级到P3）

---

## 方法 5：跨境社群/客服/直播问句校准

**来源：** 福步论坛、雨果网、卖家社群群聊、跨境客服聊天记录、直播间观众提问

**操作步骤：**
1. 从卖家社群（微信群/QQ群/Discord）收集真实提问
2. 从客服工单系统导出高频问题
3. 从直播间回放提取观众弹幕/评论中的问句
4. 从福步论坛/雨果网帖子标题收集提问

**筛选规则：**
- 只保留疑问句式（有"怎么""如何""能不能""多少"等）
- 排除闲聊/吐槽（非信息需求）
- 排除已解决的一次性问题（如"我的订单XXX怎么还没到"）
- 保留通用性强的问题（能代表一类卖家的共性需求）

**准确度判定：**
- 多人重复提问（频次 > 5）→ 80%
- 单人提问但代表性强 → 75%
- 过于个性化的问题 → 丢弃

**特别价值：** 这类问句是用户**真实话术**，AI 平台的用户也会用类似句式搜索

---

## 方法 6：谷歌 AlsoAsked / AnswerThePublic 校准

**来源：** AlsoAsked.com、AnswerThePublic.com、Google "People Also Ask"

**操作步骤：**
1. 在 AlsoAsked 输入英文词根（如"amazon seller registration"）
2. 收集所有展开的"People Also Ask"问题树
3. 在 AnswerThePublic 输入词根，收集 who/what/when/where/why/how 分类问句
4. 直接在 Google 搜索观察"People Also Ask"展开项

**适用场景：** 海外市场（NA/EU）的英文口语问句，对应 ChatGPT、Gemini 的海外检索短语

**改写规则（英文→中英双版本）：**
| 英文原句 | 保留英文版 | 生成中文版 |
|---------|-----------|-----------|
| How to register as Amazon seller from China | ✅ 保留 | "中国卖家怎么注册亚马逊" |
| What are Amazon FBA fees in 2026 | ✅ 保留 | "2026年亚马逊FBA费用是多少" |

**准确度判定：**
- 来自 Google PAA 且搜索量验证过 → 75%
- 来自 AlsoAsked 多层展开 → 72%
- 来自 AnswerThePublic 但未验证搜索量 → 68%

---

## 方法 7：AI 批量扩写（P3 兜底：句式改写 + 词根扩词）

**操作：** 用大模型对已有高分问句做句式变体改写，并用词根+句式模板批量生成补充问句

**A. 句式改写方向：**
| 原句式 | 改写为 |
|--------|--------|
| 陈述句 | 疑问句（"FBA费用很高" → "FBA费用到底有多高"） |
| 书面语 | 口语（"如何进行选品分析" → "选品怎么搞 有啥工具推荐"） |
| 长句 | 短句（拆分复合问题） |
| 中文 | 英文对应版 |
| 具体句 | 泛化句（"美国站FBA费用" → "各站点FBA费用对比"） |
| 泛化句 | 具体句（"怎么开店" → "没有公司怎么注册亚马逊"） |

**B. 词根批量扩词模板：**
```
{词根} + 如何/怎么/哪家/费用/避坑/流程/条件/区别/推荐/教程
{词根} + 新手/2026/中国卖家/第一次
{词根} + 值不值得/好不好做/有什么风险
```

**限制：**
- 只对已有高分（A/B级）短语做变体扩写
- 不对 C/D 级短语做扩写
- 每条原始短语最多生成 3 个变体
- 变体必须与原句语义不完全重复
- 生成后必须经过"意图过滤"删除不相关的
- 权重仅占总体 10%

**准确度判定：** 统一 60%（AI 生成非真实用户行为）

## 三层筛选规则（所有来源的短语统一执行）

### 第一层：意图过滤

**操作：** 对所有收集到的问句做业务相关性筛选

**保留条件（满足任一即保留）：**

**① 卖家运营类：**
- 开店/注册/审核
- FBA/物流/仓储
- 关税/VAT/税务
- Listing/产品页
- 广告/PPC/推广
- 合规/侵权/政策
- 选品/品类/趋势
- 品牌/Brand Registry
- 运营/店铺管理

**② 买家采购类：**
- 产品对比/评测
- 参数/规格/性能
- 使用场景/教程
- 购买建议/推荐
- 价格/性价比

**丢弃条件（满足任一即丢弃）：**
- 纯娱乐/八卦话题
- 竞品平台专属话题（Shopee运营、TikTok Shop）
- 与亚马逊/跨境电商完全无关的问题
- 涉及违法/违规操作的问题

### 第二层：批量召回校验（最重要）

**操作：** 把第一层通过的所有短语，批量输入 7 款大模型执行搜索

**检查内容：**
- AI 回答中是否出现我方品牌/内容/链接
- 如果完全没有我方信息 → 该短语要么是 Gap（需要生产内容），要么是无效短语（AI 不会对此引用我方）

**判定逻辑：**
```
IF 问句在 3+ 平台能触发我方内容 → 短语有效，标记 "verified"
IF 问句在 1-2 平台能触发 → 短语部分有效，标记 "partial"
IF 问句在 0 平台能触发 BUT 属于我方业务范围 → 标记为 "gap"（有内容需求但无覆盖）
IF 问句在 0 平台能触发 AND 不属于我方业务 → 丢弃
```

**注意区分：**
- `gap` = 有价值的短语，只是我们还没有对应内容 → 送智造
- `invalid` = 无效短语，即使有内容 AI 也不会引用 → 丢弃

### 第三层：去重聚类

**操作：** 对通过前两层的所有短语做语义去重和聚类

**去重规则：**
1. **完全重复：** 相同文字 → 直接删除，保留最早入库的
2. **近义重复：** 语义相似度 > 85% → 合并为一条，取更口语化/更长的版本
   - "怎么注册亚马逊" vs "亚马逊怎么注册" → 合并
   - "FBA费用多少" vs "亚马逊FBA费用怎么算" → 保留后者（更完整）
3. **同主题不同角度：** 语义相似但角度不同 → 保留两条
   - "FBA费用多少" vs "FBA费用比自发货贵吗" → 两条都保留（不同意图）

**聚类规则：**
- 按主题聚类（注册类/物流类/选品类/广告类等）
- 同一聚类内按频次排序（被多个来源产出的 = 高频 = 优先）
- 标记每条短语的来源数量：`source_count`（被几种方法产出过）

**频次加权：**
```
IF source_count >= 3 → 该短语被多个来源独立产出，高度可靠，加权 +20%
IF source_count == 2 → 两个来源交叉验证，加权 +10%
IF source_count == 1 → 单一来源，不加权
```

**操作：** 把短语直接输入 7 个 AI 平台，检查回答中是否引用我方内容

**验证维度：**

| 检查项 | 品牌类短语 | 行业类短语 |
|--------|-----------|-----------|
| 品牌提及 | ✅ 必查 | ✅ 查 |
| 官网链接 | ✅ 必查 | ⚪ 选查 |
| 内容要点 | ✅ 查 | ✅ 查 |
| 竞品出现 | ✅ 查（负面信号） | ⚪ 参考 |

**判定标准：**
- 7/7 平台都有我方内容 → 100% 覆盖（该短语不需要额外行动）
- 4-6/7 平台有 → 部分覆盖（需要优化未覆盖平台）
- 1-3/7 平台有 → 低覆盖（需要加强内容）
- 0/7 → 完全 Gap（需要全新内容）

**准确度含义：** 这个方法不验证"问句本身是否精准"，而是验证"我们对这个问句的覆盖程度"

## 方法 10：跨平台一致性验证

**操作：** 同一个问句在不同平台上，AI 回答的核心信息是否一致

**为什么需要：** 如果一个问句在不同平台得到完全不同的回答，说明该问句可能太模糊，不是好的 GEO 短语

**执行步骤：**
1. 把同一个问句输入 7 个平台
2. 提取每个平台回答的前 3 个核心要点
3. 对比要点的一致性

**判定标准：**
- 5+ 平台回答要点一致 → 高一致性（该问句有确定性答案，AI 容易标准化引用）
- 3-4 平台一致 → 中等一致性
- < 3 平台一致 → 低一致性（问句太泛或歧义，不建议作为核心短语）

**准确度评分：**
- 高一致性 → +10 分
- 中等 → +5 分
- 低一致性 → -5 分（降权）

---

## 方法 11：时效性验证

**操作：** 验证问句是否仍然活跃（不是过期的、已解决的、或季节性过期的话题）

**检查规则：**
1. 问句中是否包含年份标记？（如"2025注册流程"→ 已过期需更新为2026）
2. 该问句对应的政策/功能是否已变更？
3. 季节性问句当前是否在窗口期？（如"Prime Day备货"只在Q2-Q3有效）

**判定标准：**
- 常青话题（如"亚马逊怎么开店"）→ 时效分 100%
- 年度更新话题（如"2026 VAT新规"）→ 时效分取决于是否已更新到当前年份
- 过期话题 → 时效分 0%，标记 `is_expired = TRUE`

**自动检测规则：**
```
IF 问句包含年份 AND 年份 < 当前年份 → 标记为过期
IF 问句涉及已下线功能/已取消政策 → 标记为过期
IF 问句为季节性 AND 当前不在窗口期 → 标记为"待激活"
```

---

## 方法 12：意图匹配验证

**操作：** 验证问句的用户意图是否与我方能提供的内容类型匹配

**意图分类：**
| 意图类型 | 我方是否有对应内容类型 | 示例 |
|---------|---------------------|------|
| 信息查询 (informational) | ✅ 科普文章、指南 | "亚马逊FBA是什么" |
| 操作指导 (transactional) | ✅ 步骤教程、流程文 | "怎么注册亚马逊卖家账号" |
| 对比选择 (comparison) | ✅ 对比文、选品指南 | "亚马逊美国站vs欧洲站哪个好" |
| 费用咨询 (cost) | ✅ 费用详解、成本分析 | "亚马逊开店要多少钱" |
| 售后投诉 (complaint) | ❌ 我方不生产此类内容 | "亚马逊卖家被封号怎么申诉" |
| 竞品评测 (competitor) | ❌ 我方不涉及竞品 | "Shopee和亚马逊哪个好" |

**判定标准：**
- 意图匹配（前4类）→ 意图分 100%
- 意图边缘（如投诉类但可以转化为合规指南）→ 意图分 60%
- 意图不匹配（竞品类、纯负面类）→ 意图分 0%，标记为丢弃

## 综合准确度评分公式

```
综合准确度 = 
    来源基础分 × 0.25          (来源本身的可靠度: P1=92-95%, P2=75-85%, P3=60%)
  + 逆向召回共识度 × 0.20      (7平台中几个认为能命中)
  + 下拉联想命中度 × 0.15      (是否出现在平台下拉建议中)
  + 正向覆盖率 × 0.15          (智测验证：能在几个平台触发我方内容)
  + 跨平台一致性 × 0.08        (各平台对同一问句回答是否一致)
  + 时效性分 × 0.07            (是否过期)
  + 意图匹配分 × 0.05          (是否与我方业务匹配)
  + 来源交叉验证加权 × 0.05    (被几种来源独立产出: source_count)
```

**去重后的频次加权（额外加分）：**
```
source_count >= 3 → 综合分 × 1.20
source_count == 2 → 综合分 × 1.10
source_count == 1 → 不加权
```

**评分等级：**
| 综合分 | 等级 | 行动 |
|--------|------|------|
| 85-100 | A（高精准） | 核心短语，优先生产内容 |
| 70-84 | B（可用） | 标准短语，正常流程 |
| 50-69 | C（需验证） | 需要更多交叉验证或优化 |
| < 50 | D（低质量） | 降级到 P3 或丢弃 |
